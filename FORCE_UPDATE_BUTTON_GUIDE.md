# 强制更新按钮使用说明

## 按钮位置

强制更新按钮位于 **任务栏** 中，在上传按钮旁边。

### 任务栏位置
- 屏幕底部
- 鼠标移到屏幕底部边缘时自动显示
- 可以点击图钉按钮固定任务栏

### 按钮图标
🔄 **刷新图标** - 圆形箭头

### 按钮顺序（从左到右）
1. 📌 固定任务栏
2. 中 中文输入
3. 🔒 修改密码
4. ⬆️ 上传文件
5. **🔄 强制更新** ← 新增
6. 连接状态指示器

## 使用步骤

### 1. 打开更新界面
- 将鼠标移到屏幕底部，显示任务栏
- 点击 **🔄 刷新图标** 按钮
- 弹出"强制更新"对话框

### 2. 检查版本
对话框会自动检查更新，显示：
```
当前版本: 0.2.0
最新版本: 0.3.0
⚠️ 更新过程中服务将短暂中断
```

如果已是最新版本：
```
当前版本: 0.2.0
✓ 已是最新版本
```

### 3. 开始更新
- 点击 **"开始更新"** 按钮
- 确认对话框：点击 **"确定"**
- 更新开始

### 4. 查看进度
更新过程中会显示：

**进度条**：
```
更新进度 ████████░░ 80%
```

**实时日志**：
```
[0/10] 保存运行中的应用状态...
[0/10] 应用状态已保存
[1/10] 检测系统架构...
[1/10] 系统架构: amd64
[2/10] 准备下载最新版本...
[3/10] 下载中... 5/10 MB
[3/10] 下载完成
[4/10] 验证下载文件...
[5/10] 备份当前版本...
[6/10] 设置执行权限...
[7/10] 替换程序文件...
[8/10] 清理临时文件...
[9/10] 准备重启服务...
[10/10] 重启服务...
```

### 5. 等待重启
更新完成后：
```
更新完成，等待服务重启...
服务已恢复，即将刷新页面...
```

页面会自动刷新，连接到新版本。

## 日志颜色说明

- 🔵 **蓝色** - 信息提示
- 🟢 **绿色** - 成功消息
- 🔴 **红色** - 错误消息
- 🔷 **天蓝色** - 进度更新

## 更新失败处理

如果更新失败：
1. 日志会显示红色错误信息
2. 系统会自动回滚到旧版本
3. 点击 **"关闭"** 按钮退出
4. 可以重试或联系管理员

## 注意事项

### ⚠️ 更新期间
- 服务会短暂中断（通常 < 30 秒）
- 所有连接会断开
- 不要关闭浏览器
- 不要刷新页面

### ✅ 更新后
- 页面会自动刷新
- 之前运行的 Pake apps 会自动恢复
- 检查版本号确认更新成功

### 🔒 权限要求
- 需要管理员权限（如果启用了 Basic Auth）
- 服务器需要有写入权限

## 快捷访问

### 方法 1：任务栏按钮（推荐）
1. 鼠标移到屏幕底部
2. 点击 🔄 刷新图标

### 方法 2：API 直接访问
```bash
# 检查版本
curl http://localhost:8080/api/version

# WebSocket 更新（需要认证）
websocat ws://localhost:8080/api/upgrade/ws
```

## 故障排除

### 问题 1：找不到更新按钮
**解决**：
- 确保鼠标移到屏幕最底部
- 任务栏会自动显示
- 或点击图钉固定任务栏

### 问题 2：点击按钮无反应
**解决**：
- 检查浏览器控制台是否有错误
- 刷新页面重试
- 检查网络连接

### 问题 3：更新失败
**解决**：
- 查看日志中的错误信息
- 检查磁盘空间是否足够
- 检查网络连接到 GitHub
- 查看服务器日志：`journalctl -u ivnc -n 50`

### 问题 4：更新后服务未恢复
**解决**：
- 等待 30 秒
- 手动刷新页面
- 检查服务状态：`systemctl status ivnc`
- 手动重启：`systemctl restart ivnc`

## 技术细节

### 更新流程
1. **检查版本** - 从 GitHub API 获取最新版本
2. **下载** - 从 GitHub Releases 下载对应架构的二进制文件
3. **验证** - 检查文件大小和完整性
4. **备份** - 备份当前版本（带时间戳）
5. **替换** - 原子替换二进制文件
6. **重启** - 优先使用 systemd，失败则 exec 自我重启
7. **恢复** - 自动恢复之前运行的 Pake apps

### 支持的架构
- ✅ x86_64 (amd64)
- ✅ aarch64 (arm64)

### 下载源
```
https://github.com/Xiechengqi/iVnc/releases/download/latest/ivnc-linux-amd64
https://github.com/Xiechengqi/iVnc/releases/download/latest/ivnc-linux-arm64
```

### 备份位置
```
/usr/local/bin/ivnc.backup-1709654321
```

### 状态文件
```
~/.config/ivnc/app_running_state.json
```
